# User Case Rubric

Use this rubric when validating new tutor output against real user transcripts.

## Core dimensions

1. **Knowledge closure**
   - Does a single tutor reply leave the learner with a stable takeaway?
2. **Teach quality**
   - When the learner does not know, does the system explain the mechanism clearly instead of only reframing the question?
3. **Conversation efficiency**
   - Does the system avoid unnecessary repeated probing on the same unit?
4. **Confirmation quality**
   - When the learner is partly right, does the tutor explicitly acknowledge the valid part before refining?
5. **Naturalness**
   - Does the visible reply read like a tutor, not a structured diagnostic report?
6. **Revisit readiness**
   - If the point is deferred, is there a clear reason and a meaningful takeaway to revisit later?

## Fail conditions

- The tutor asks multiple near-duplicate questions after the learner says "不知道"
- The tutor relies on analogy without giving the actual mechanism
- The learner cannot extract a one-sentence takeaway from the reply
