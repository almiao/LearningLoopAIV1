# 架构拆分需求文档
**项目：AI 面试辅导平台**
**当前状态：单进程 Node.js 前后端一体**
**目标：前后端分离，后端引入 Python AI 服务**

---

## 背景与目标

当前应用是一个单进程 Node.js 应用，前端静态资源与后端 API 路由混合在同一进程中。随着 AI 面试功能的引入，需要将架构拆分为：

- **前端**：独立部署的 Next.js 应用（Web 优先）
- **BFF（Backend for Frontend）**：精简后的 Node.js 层，负责认证、会话、非 AI 业务路由
- **AI 服务**：独立 Python FastAPI 服务，负责面试 Agent、LLM 调用、流式输出

拆分后三者独立部署、独立扩缩容，互相通过 HTTP/SSE 通信。

---

## 目标架构

```
浏览器
  │  HTTPS
  ▼
Next.js 前端（独立部署）
  │  REST / SSE
  ▼
Node.js BFF（精简版原应用）
  ├── 用户认证 / 会话管理
  ├── 题库 CRUD、用户数据
  └── 代理转发 AI 请求
        │  HTTP / SSE
        ▼
      Python FastAPI（AI 服务）
        ├── 面试 Agent（LangGraph）
        ├── LLM 调用（Claude / GPT）
        └── 流式输出 SSE
```

---

## 拆分范围

### 需求 1：剥离前端为独立 Next.js 应用

**背景**
当前前端页面由 Node.js 直接 serve，模板或静态资源与服务器代码耦合。

**验收标准**
- 前端迁移至独立 Next.js 项目，可独立 `npm run dev` 启动
- 所有页面路由、组件、样式、静态资源从原应用中移出
- 前端通过环境变量 `NEXT_PUBLIC_API_BASE_URL` 指向 BFF 地址
- 本地开发：前端 `:3000`，BFF `:4000`，跨域请求正常（BFF 配置 CORS）
- 原应用删除所有 `res.render()`、`express.static()` 等前端 serve 逻辑

**不在范围内**
- UI 改版、组件库迁移
- SSR/SSG 优化（首期 CSR 即可）

---

### 需求 2：精简 Node.js BFF，规范 API 边界

**背景**
原应用混合了业务逻辑、数据库操作、路由处理。拆分后 BFF 只保留必要职责，AI 相关逻辑全部移至 Python 服务。

**BFF 保留职责**
- 用户注册 / 登录 / JWT 鉴权
- 用户档案 CRUD（姓名、目标岗位、历史记录摘要）
- 题库管理（增删改查，非 AI 生成部分）
- 面试历史记录读写
- 代理转发：将 `/api/interview/*` 的请求透传至 Python AI 服务，并转发 SSE 流

**BFF 移除内容**
- 对话引擎
- 所有直接调用 LLM 的代码
- 所有 prompt 拼接逻辑
- 所有 AI 评分、追问生成逻辑


**验收标准**
- BFF 启动后不含任何 LLM 相关依赖（`openai`、`@anthropic-ai/sdk` 等从 package.json 移除）
- 所有 `/api/interview/*` 请求均被正确代理，SSE 流不中断、不丢包
- JWT 鉴权中间件覆盖所有需要登录的路由
- BFF 单独可测试（`jest` 或 `supertest`）

---

### 需求 3：新建 Python FastAPI AI 服务

**背景**
AI 面试逻辑全部在此服务中实现，BFF 对其完全透明，只做请求转发。

**技术选型**
- 运行时：Python 3.11+
- Web 框架：FastAPI + uvicorn
- LLM：迁移当前设置
- 流式输出：FastAPI `StreamingResponse` + Server-Sent Events



**(c) 状态持久化**

- 会话状态（当前题号、对话历史、评分累积）存 Redis（TTL 2 小时）
- 最终报告写入 PostgreSQL（由 BFF 数据库共享，AI 服务只写报告表）

---

### 需求 4：本地开发环境联调

**背景**
拆分后三个进程需要在本地同时运行并正确通信。

**端口约定**

| 服务 | 本地端口 |
|------|----------|
| Next.js 前端 | 3000 |
| Node.js BFF | 4000 |
| Python AI 服务 | 8000 |

**环境变量**

前端 `.env.local`：
```
NEXT_PUBLIC_API_BASE_URL=http://localhost:4000
```

BFF `.env`：
```
PORT=4000
JWT_SECRET=dev-secret
AI_SERVICE_URL=http://localhost:8000
DATABASE_URL=postgresql://...
REDIS_URL=redis://localhost:6379
```

Python AI 服务 `.env`：
```
PORT=8000
ANTHROPIC_API_KEY=sk-ant-...
REDIS_URL=redis://localhost:6379
DATABASE_URL=postgresql://...
```

**验收标准**
- 提供 `docker-compose.yml` 启动 PostgreSQL 和 Redis
- 提供根目录 `Makefile` 或 `start-dev.sh`，一条命令同时启动三个服务
- README 中记录完整的本地启动步骤
